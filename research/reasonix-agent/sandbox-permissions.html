沙箱与权限模型

Reasonix 的“沙箱”是应用层安全边界：项目根目录约束、命令 allowlist、敏感路径降级、确认闸门和 edit gate。它没有实现 OS 级隔离。

边界判断

如果把“沙箱”理解为 macOS Seatbelt、Linux Landlock/seccomp、container namespace 或虚拟机隔离，Reasonix 当前没有这类实现。它的工具进程仍以当前用户身份运行。

如果把“沙箱”理解为 agent 工具层边界，Reasonix 做得相当细：路径解析默认钉在 workspace root，外部路径需要确认，shell 命令逐 segment tokenize 和 allowlist，危险参数和敏感路径会让命令降级为需确认。

结论：Reasonix 的安全模型适合“本地可信用户 + 明确确认高风险动作”，不适合被当成可公开暴露的强隔离执行环境。

文件系统：rootDir、safePath 与外部路径审批

文件工具以 rootDir 为边界。相对路径、以 / 开头的项目绝对路径、以及 OS 绝对路径都会进入统一的 safePath 解析流程。

  • 普通相对路径必须解析到 rootDir 内，否则报 sandbox escape。
  • OS 绝对路径如果在 rootDir 外，会触发 path_access 确认。
  • 确认支持 deny、run_once、always_allow。always_allow 会把路径前缀写入项目配置。
  • 文件读取、目录列举、搜索等 read-only 工具可 parallel-safe；写入工具需要额外经过写 gate。

外部路径的 always_allow 是便利性功能，也是权限扩大点。对 ~/ 或上层目录做持久允许，会明显弱化 root sandbox 的意义。

Shell 命令：不走系统 shell，但仍是用户进程

run_command 和 run_background 的 cwd pinned 到 rootDir。底层执行使用 spawn(..., { shell: false })，并通过 tokenizer 解析 argv，避免直接交给 shell 展开环境变量、glob、反引号或命令替换。

Reasonix 内置一组 safe-ish allowlist，例如只读 git、ls/pwd/cat/head/tail/wc/file/tree/find/grep/rg、版本查询、常见测试和 lint 命令。命令链会逐 segment 检查，pipe、逻辑运算、重定向都会被解析，heredoc 和后台 & 被拒绝。

Allowlist prefix

命令按 argv prefix 匹配，而不是按原始字符串包含判断。

Risky args

find -delete、find -exec、eslint --fix、ruff --fix 等会让命令降级为需确认。

Sensitive paths

~/.ssh、~/.aws、*.env、*.pem、secret/credential 相关路径会触发敏感路径判断。

Redirect checks

重定向目标必须留在 sandbox 内，并做 realpath/symlink escape 检查。

编辑闸门：read-before-edit 与 review/auto/yolo

文件编辑不是直接写。edit_file 和 multi_edit 要求目标文件在本 session 中已读过，否则拒绝。单文件编辑要求 SEARCH 块精确唯一匹配；multi-edit 会先验证所有 edits，再进入写阶段，失败时尝试回滚已写文件。

用户配置中的 editMode 是主要信任拨盘：

模式 含义 安全影响
review 编辑进入 review gate，shell 仍需 gate。 默认较稳，适合日常使用。
auto 自动应用编辑，但 shell 仍需 gate。 提高效率，但文件修改更依赖模型判断。
yolo 跳过编辑和 shell gate。 不适合含敏感凭据或高价值代码库的机器。
plan 阻止所有非 read-only 工具。 适合只做设计、审阅和探索。

模式与配置：权限是可持久化的

项目配置会保存 shell allowlist、hooksTrusted、pathAllowed、dashboard 和模型偏好。PauseGate 是工具层与 UI 层的桥：shell、path、plan、choice、checkpoint、revision 等确认请求都会通过它发起和解析。

这意味着权限审批既能在 TUI 中完成，也能被 dashboard 或远程通道承接。功能上很顺滑，但也要求远程通道本身不能被随意暴露。

风险矩阵

风险 Reasonix 现有缓解 剩余风险
模型执行破坏性命令 allowlist、危险参数降级、PauseGate、plan mode。 用户批准或 yolo 后，命令仍以当前用户身份运行。
读取敏感文件 rootDir 限制、外部路径审批、敏感路径检测。 持久允许宽路径后，模型可继续访问该路径前缀。
命令注入 shell:false、argv tokenize、拒绝 heredoc/backticks/substitution。 允许的可执行文件自身仍可能执行二级逻辑。
dashboard 被远程滥用 token gate、mutation header、非 loopback 警告。 无内置 TLS；token 泄漏即相当于控制面暴露。

部署建议

  • 默认保持 review 或 auto，只在临时、低风险目录使用 yolo。
  • 不要把宽路径加入 pathAllowed。优先批准具体文件或具体子目录。
  • 远程访问 dashboard 时优先用 SSH tunnel 或可信 LAN，不建议直接把 host 绑到公网地址。
  • 在含生产凭据的机器上，把 Reasonix 当成本地工具，而不是强隔离执行沙箱。
