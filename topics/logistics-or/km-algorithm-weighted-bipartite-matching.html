KM 算法：顶标、相等子图与最大权二分图匹配

KM 算法用于求解最大权完美匹配。它可以理解为：在普通二分图匹配的增广逻辑之上，增加一套“顶标 / 价格体系”，逐步构造相等子图；当相等子图中存在完美匹配时，这个匹配就是最大权匹配。

1. 问题形式：最大权完美匹配

给定左右两侧各 n 个节点，任意左侧节点和右侧节点之间有一个权重。目标是从中选出 n 条边，使每个左点和每个右点都刚好出现一次，并让总权重最大。

输入：weight[i][j] = 左侧点 i 匹配右侧点 j 的收益 约束：每个左点匹配一个右点；每个右点也只能被一个左点匹配 目标：maximize sum(weight[i][match[i]])

这类问题也可以称为指派问题的最大收益版本。若问题是最小成本，可以通过取负数或用常数转换成最大收益形式。

2. 顶标、相等边与相等子图

KM 算法会给左右两侧的点各设置一个数：

左侧顶标：lx[i] 右侧顶标：ly[j]

它始终维护一个条件：

lx[i] + ly[j] >= weight[i][j]

如果一条边满足等号：

lx[i] + ly[j] == weight[i][j]

这条边就是相等边。所有相等边构成的二分图，就是相等子图。

直觉：顶标是一套算法里的“估价体系”。相等边表示在当前估价体系下“刚好贴边”的候选边。KM 只在这些候选边里尝试寻找完美匹配；如果找不到，就调整顶标，让更多边进入相等子图。

3. 3×3 示例完整推导

使用一个最大权匹配例子：

人 / 任务T1T2T3
A346
B235
C164

3.1 初始化顶标

通常初始化为：

lx[i] = 第 i 行最大权重 ly[j] = 0

所以：

lx[A] = 6, lx[B] = 5, lx[C] = 6 ly[T1] = 0, ly[T2] = 0, ly[T3] = 0

3.2 初始相等边

检查哪些边满足 lx[i] + ly[j] == weight[i][j]：

节点相等边说明
AA - T36 + 0 = 6
BB - T35 + 0 = 5
CC - T26 + 0 = 6
初始相等子图： A - T3 B - T3 C - T2

这个相等子图中找不到完美匹配，因为 A 和 B 都只能连接 T3，冲突无法化解。

3.3 第一次顶标调整

假设当前匹配中 A-T3、C-T2 已经存在，B 还未匹配。B 想要 T3，但 T3 被 A 占用，于是搜索树为：

S = {B, A} T = {T3}

计算从 S 中左点到 T 外右点的 slack：

slack[i][j] = lx[i] + ly[j] - weight[i][j] B-T1: 5 + 0 - 2 = 3 B-T2: 5 + 0 - 3 = 2 A-T1: 6 + 0 - 3 = 3 A-T2: 6 + 0 - 4 = 2 delta = min(slack) = 2

调整规则：

S 中左点：lx 减 delta T 中右点：ly 加 delta 其他点不变

调整后：

lx[A] = 4, lx[B] = 3, lx[C] = 6 ly[T1] = 0, ly[T2] = 0, ly[T3] = 2

新的相等边：

A - T2, A - T3 B - T2, B - T3 C - T2

此时仍然找不到完美匹配，因为所有相等边只连到了 T2、T3，没有连到 T1。

3.4 第二次顶标调整

继续从未匹配点 B 出发。B 可以连 T2、T3，但 T2 被 C 占，T3 被 A 占，所以搜索树扩展为：

S = {A, B, C} T = {T2, T3}

不在 T 中的右点只有 T1。计算：

A-T1: 4 + 0 - 3 = 1 B-T1: 3 + 0 - 2 = 1 C-T1: 6 + 0 - 1 = 5 delta = 1

调整后：

lx[A] = 3, lx[B] = 2, lx[C] = 5 ly[T1] = 0, ly[T2] = 1, ly[T3] = 3

新的相等子图变成：

A - T1, T2, T3 B - T1, T2, T3 C - T2
现在可以找到完美匹配。例如：C-T2、A-T3、B-T1，总权重为 6 + 6 + 2 = 14。

4. 相等子图里的匹配如何确定？

第二次调整后，相等子图是：

A: T1, T2, T3 B: T1, T2, T3 C: T2

这里 C-T2 看起来像是“确定”的，因为 C 只有 T2 一条相等边。如果最终要形成覆盖全部节点的完美匹配，C 必须匹配 T2。

一旦 C-T2 被占用，剩下的是：

左侧：A, B 右侧：T1, T3

A 和 B 都可以连 T1、T3，所以可能得到：

方案一：A-T3, B-T1, C-T2，总权重 = 6 + 2 + 6 = 14 方案二：A-T1, B-T3, C-T2，总权重 = 3 + 5 + 6 = 14
关键澄清：KM 不是直接“看出来”A-T3、B-T1。真正做法是在相等子图上重新执行普通二分图匹配 / 增广路径搜索。只要找到覆盖所有左右节点的 n 条不冲突边，就是完美匹配。

4.1 不是任意不冲突匹配都叫完美匹配

要特别区分：

匹配

一组不冲突的边。每个点最多被一条边使用。

最大匹配

在当前图中，边数最多的一组不冲突边。

完美匹配

如果左右各 n 个点，匹配边数 = n，并且所有点都被覆盖。

KM 的终止条件

相等子图中的最大匹配数 = n，也就是找到完美匹配。

例如相等子图只有：

A - T1 B - T1 C - T2

可以找到不冲突匹配 A-T1、C-T2，但只有 2 条边。若左右各 3 个点，这不是完美匹配，KM 还不能结束。

4.2 节点更多时怎么计算？

节点更多时，不靠肉眼判断，而是在相等子图上跑普通二分图最大匹配：

如果最大匹配数 == n： 找到完美匹配，KM 结束 否则： 相等边还不够，继续调整顶标

也就是说，KM 的内部仍然在使用你已经熟悉的“增广路径 / 让路”逻辑，只是它使用的边不是原始全部边，而是当前顶标体系下的相等边。

5. 为什么相等子图中的完美匹配一定最优？

KM 维护：

lx[i] + ly[j] >= weight[i][j]

对于任意一个完美匹配，因为每个左点和每个右点都刚好被使用一次，所以：

任意匹配总权重 <= sum(lx) + sum(ly)

这意味着 sum(lx) + sum(ly) 是当前能够证明出来的一个总收益上界。

如果某个完美匹配完全由相等边组成，那么每条匹配边都满足：

weight[i][j] = lx[i] + ly[j]

于是这组匹配的总权重正好等于：

sum(lx) + sum(ly)
它达到了理论上界，因此一定是最大权完美匹配。

6. KM 算法与行列变换版匈牙利算法的关系

如果你已经理解了行列变换版匈牙利算法，会发现它和 KM 算法有很强的相似性。两者都不是直接贪心选边，而是先维护一套“变换后的评价体系”，再从当前看起来最有希望的边里寻找完整匹配。

行列变换版匈牙利算法是在矩阵里制造 0；KM 算法是在二分图里制造相等边。

两种讲法的入口不同：行列变换版通常从最小成本矩阵出发，KM 通常从最大权二分图出发。但它们背后的共同思想，都是通过调整对偶变量，把“可能属于最优解的边”逐渐暴露出来。

行列变换版匈牙利算法KM 算法直觉对应
成本矩阵权重矩阵都是描述左侧对象和右侧对象之间的匹配价值
行减、列减调整左右顶标都在改变评价体系，而不是改变原问题的最优结构
0 元素相等边当前评价体系下最值得尝试的候选边
由 0 构成的可选图相等子图算法只在这些候选边中寻找完整匹配
从 0 中选不冲突指派在相等子图中找完美匹配本质上都是普通二分图匹配问题
0 不够时继续调整矩阵无法增广时用 slack 调整顶标候选边不足时，继续制造新的候选边

所以可以把 KM 看成行列变换思想在加权二分图上的一种图论表达：矩阵算法用“0”提示哪些边可选，KM 用“相等边”提示哪些边可选；矩阵算法通过覆盖 0 后调整矩阵制造新 0，KM 通过最小 slack 调整顶标制造新相等边。

差异也要注意：行列变换版更适合教学最小成本指派，矩阵操作直观；KM 更适合从最大权匹配、对偶上界和增广路径的角度理解，也更容易和图算法实现衔接。

7. KM 算法流程

  1. 初始化顶标。通常令每个左点顶标等于该行最大权重，右点顶标为 0。
  2. 构造相等子图。只保留满足 lx[i] + ly[j] == weight[i][j] 的边。
  3. 在相等子图中找匹配。使用普通二分图匹配的增广路径逻辑。
  4. 如果找到 n 条不冲突边。说明相等子图中存在完美匹配，算法结束。
  5. 如果找不到。根据当前搜索树计算最小 slack，也就是 delta。
  6. 调整顶标。S 中左点 lx 减 delta，T 中右点 ly 加 delta，其他不变。
  7. 产生新相等边。继续在扩大的相等子图中寻找增广路径，直到形成完美匹配。

一句话总结

KM 算法 = 普通二分图匹配 + 顶标调整。
普通匹配负责在当前候选边里找增广路径；顶标调整负责在候选边不够时，制造新的相等边。
