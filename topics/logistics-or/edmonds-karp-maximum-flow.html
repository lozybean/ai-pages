Edmonds-Karp 算法：用 BFS 稳定选择增广路径

Edmonds-Karp 是 Ford-Fulkerson 方法的一种确定性实现：每一轮都在残量网络里用 BFS 找边数最少的增广路径。它不改变最大流的建模方式，却把“随便找路”的不稳定性变成了有复杂度上界的可控过程。

一、它解决什么问题

Edmonds-Karp 用来求有向容量网络中的最大流。给定源点 s、汇点 t 和每条边的容量 c(u, v)，目标是在满足容量约束和流量守恒的前提下，让从 s 流向 t 的总流量最大。

容量约束：0 <= f(u, v) <= c(u, v) 流量守恒：除 s、t 外，流入 = 流出 目标函数：maximize |f|
Edmonds-Karp 的本质不是发明新的最大流模型，而是规定每次都走残量网络中的最短增广路径。

这里的“最短”不是容量最大、距离最短或成本最低，而是边数最少。这正好是 BFS 擅长找到的路径。

二、核心思想

Ford-Fulkerson 的基本框架是：只要残量网络中还存在从 s 到 t 的增广路径，就沿这条路径增加瓶颈流量。问题是，如果“任意选择路径”，算法可能走很多绕路，甚至在非整数容量上出现麻烦。

朴素选择

每次随意找一条可行路径。路径可能很绕，早期选择可能不断制造后续调整。

Edmonds-Karp

每次 BFS 找边数最少的增广路径。选择规则固定，路径层数会形成单调推进的结构。

残量网络仍然包含两类边：

残量边残量含义
原边 u -> vc(u, v) - f(u, v)还可以继续往前增加多少流量。
反向边 v -> uf(u, v)最多可以撤销多少已经走过的真实流量。
所以 Edmonds-Karp 的一句话版本是：在残量网络上反复 BFS，找到最少边数的 s -> t 路径，沿路径推入瓶颈流量，直到 BFS 找不到 t。

BFS 为什么一定找到边数最少的增广路径

在 Edmonds-Karp 里，残量网络的每条可走边都只算“一步”，没有边权差异。BFS 的队列按层推进：先处理离 s 0 条边的点，再处理 1 条边能到的点，再处理 2 条边能到的点。一个点第一次被访问时，它的层数就是从 s 到它的最少边数。

第 0 层：s 第 1 层：从 s 走 1 条残量边能到的点 第 2 层：从 s 走 2 条残量边能到、且之前没到过的点 ... 第一次发现 t 时：得到的就是边数最少的 s -> t 增广路径

原因很朴素：如果存在一条更短的路径，它的上一层节点一定会更早出队，并在当前路径之前发现 t。所以 BFS 不是“碰巧找到短路”，而是用队列顺序保证先穷尽所有更短路径。

三、算法流程

完整流程可以写成下面几步：

初始化流量为 0 -> 构造残量网络 -> BFS 找最短增广路径 -> 按瓶颈增广 -> 重复直到不可达

伪代码如下：

max_flow = 0
while BFS can reach t in residual graph:
    P = path restored by parent pointers
    delta = min residual_capacity(e) for e in P
    for each edge e in P:
        if e is a forward residual edge:
            increase original flow on e by delta
        else:
            decrease original flow on the reverse original edge by delta
    max_flow += delta
return max_flow

BFS 的父指针不仅用来判断 t 是否可达，也用来还原这一轮到底选择了哪条最短增广路径。

四、示例网络

下面用一个 6 个点的容量网络完整推导。源点 s 向 A 和 B 供给，汇点 t 从 C 和 D 接收；中间边决定最终能通过多少。

所有边的初始流量都是 0，因此初始残量等于容量。我们假设每个点的邻接边按图中常见阅读顺序扫描：从 s 先看 A 再看 B，从 A 先看 C 再看 D，最后再看 A -> B。

图里有一条 A -> B 容量为 2 的边。它不会改变下面的推导结果：初始 BFS 中 B 已经通过 s -> B 在第 1 层被发现，A -> B 不会给 B 更短路径；后面 s -> A 满流后，A 从源点侧不可达，这条边也就无法继续帮助到达 t。残量图中会把它弱化标成 A->B 正 2。

六、终止与最小割

第 4 轮之后，再从 s 做 BFS。s -> A 正向残量为 0，s -> B 还剩 1，所以 B 可达；但 B -> D 正向残量已经为 0，无法继续进入 D。因此残量网络中从 s 可达的点只有：

S = {s, B} T = {A, C, D, t}

这个可达集合自然给出一个割。原图中从 S 指向 T 的边只有两条：

跨割边容量最终状态
s -> A10满流
B -> D9满流
cut(S, T) = c(s, A) + c(B, D) = 10 + 9 = 19 当前流量 |f| = 19
当前流量等于一个割的容量。因为任意流都不会超过任意割容量，所以当前流已经是最大流。

七、复杂度为什么稳定

Edmonds-Karp 的复杂度是：

O(V * E^2)

直觉上可以这样理解：

每轮 BFS

在残量网络上扫描边，代价是 O(E)。

增广轮数

每次某条边成为瓶颈后，如果未来还想再次作为关键边出现，它到源点的 BFS 层数必须增加。这样的层数增加次数受 V 限制。

更形式化地说，BFS 保证残量网络中点到源点的最短距离不会因为一次增广而随意下降；每条边被“关键饱和”的次数是有限的，整体增广次数可界定在 O(VE)，乘上每轮 BFS 的 O(E)，得到 O(VE^2)。

这个上界通常比朴素 Ford-Fulkerson 的 O(E * F) 更稳定，因为它不依赖最大流值 F 的数值大小。

八、实现要点

  • 用邻接表存残量边，每条边都配一条反向边，更新时同时改正反两条边的残量。
  • BFS 只走残量大于 0 的边，并记录父节点、父边和路径瓶颈。
  • 如果容量是整数，最大流一定是整数；Edmonds-Karp 即使容量较大，也不会像朴素路径选择那样受 F 直接控制。
  • 工程上如果图很大，常用 Dinic 获得更好的实践性能；但理解 Edmonds-Karp 是掌握 Dinic 分层图思想的很好前置。
