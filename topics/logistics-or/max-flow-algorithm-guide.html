最大流算法：从残量网络到业务建模

最大流不是在一张图里找一条最好的路，而是在容量受限的网络中安排一组可以同时存在的流动方案。理解残量网络、反向边和最大流最小割定理，就能把物流履约、库存分配、服务容量和匹配问题统一到同一种建模语言里。

一、最大流在问什么

最大流问题研究的是：在一个有方向、有容量上限的网络中，从源点 s 到汇点 t 最多可以通过多少流量。

最大流不是找一条最宽的路，而是找一组互相协调的通路，让整个网络的总通过量最大。
直觉图景

把网络看成水管系统。每条边是一根管道，容量是管径上限；源点是水源，汇点是出口。最大流就是整个系统最多能排出多少水。

业务图景

把网络看成仓网、运力、服务调用或资源分配系统。边容量表示可运输量、处理量、库存量或可匹配次数，最大流表示系统在当前约束下的满足上界。

供给端
产能 / 库存 -> 中间网络
路由 / 转运 / 处理能力 -> 需求端
订单 / 请求 / 任务 -> 最大可满足量

这类问题的关键不是局部路径看起来多顺，而是多个路径同时使用时会不会争抢同一条瓶颈边。最大流算法正是为了处理这种全局容量协调。

二、流网络的数学模型

一个流网络通常记为有向图：

G = (V, E)

其中 V 是节点集合，E 是有向边集合。每条边 (u, v) 有非负容量 c(u, v)，并指定源点 s 和汇点 t。

1. 流函数

对每条边定义实际通过的流量：

f(u, v) = 边 u -> v 上的实际流量

2. 三个核心约束

约束公式含义
容量约束0 <= f(u, v) <= c(u, v)每条边不能超过自己的承载上限。
流量守恒sum f(u, v) = sum f(v, w)除源点和汇点外，中间节点流入多少就流出多少。
最大化目标max |f|让从源点流出的总量，也就是进入汇点的总量最大。

3. 流量值

一个可行流的值可以从源点出边计算，也可以从汇点入边计算：

|f| = sum f(s, v) = sum f(u, t)

两者相等来自流量守恒：中间节点只是转运，不会凭空产生或吞掉流量。

三、为什么需要残量网络

最朴素的想法是：只要还能找到一条从 s 到 t 的路，就沿着这条路尽量送流。路径能送多少，由路径上最小剩余容量决定。

bottleneck = min(5, 2, 4) = 2

但最大流不是简单地把每条路径依次填满。早期流量可能占用关键边，后续需要调整已有安排。残量网络就是用来表达“还能增加多少”和“已经安排的流量最多能撤回多少”。

1. 反向边不是业务倒流，而是算法反悔

假设原图有一条边：

u -> v, capacity = 10, current flow = 6

这时正向还能继续增加 4，而已经送出的 6 可以在算法上被撤回。因此残量网络中同时存在：

u -> v, residual = 4 v -> u, residual = 6
容易误解的点

反向边不表示真实业务中货物、订单或请求真的反向流动。它表示算法可以取消过去在 u -> v 上安排的一部分流量，把容量释放给更合适的组合。

2. 增广路径是在残量网络中找调整机会

如果残量网络里仍然存在从 s 到 t 的路径，就说明当前流还可以通过“继续增加”和“局部撤销”组合起来变得更大。这条路径称为增广路径。

沿增广路径增加 delta： 正向残量 -= delta 反向残量 += delta 总流量 += delta

当路径中走到某条反向边时，含义就是减少对应原边上的流量。这一步让最大流算法区别于简单贪心分配：它不是一次性占坑，而是在不断修正中逼近全局最优。

3. 概念链路

最大流算法的核心链路是：容量约束决定每条边的上限；残量网络描述当前还能怎么调整；增广路径带来新的流量增长；当残量网络中已经没有增广路径时，当前流达到最大。至于“为什么某些路径选择需要反悔”的完整推导，更适合放在下一篇 Ford-Fulkerson 中展开。

四、最大流最小割定理

最大流算法为什么正确，最终依赖一个核心定理：最大流的值等于最小割的容量。

1. 什么是割

把所有节点分成两个集合 S 和 T，满足 s in S 且 t in T。所有从 S 指向 T 的边构成一个割，割容量是这些边容量之和：

capacity(S, T) = sum c(u, v), 其中 u in S, v in T

2. 为什么任何流都不会超过任何割

从源点到汇点的所有流量都必须穿过任意一个割。割上的边容量合起来就是这道边界允许通过的最大上限，所以：

任意可行流 <= 任意割容量 最大流 <= 最小割

3. 没有增广路径时意味着什么

当残量网络中已经无法从 s 到达 t 时，说明无论是继续使用剩余容量，还是撤销并重排已有流量，都不能让总流量继续增加。此时从 s 仍可达的点和不可达的点之间，会形成一道被容量卡住的边界。

找不到增广路径，不只是“算法搜不到路了”，而是当前网络已经出现了一道容量刚好等于当前流量的最小割。

五、三类经典算法

最大流算法的差异主要在两件事：如何找增广路径，以及一轮增广能推进多少流量。

1. Ford-Fulkerson 方法

Ford-Fulkerson 是一类方法框架：不断在残量网络中寻找增广路径，并沿路径推入瓶颈流量。

找增广路径 -> 计算瓶颈容量 -> 沿路径增广 -> 更新残量网络

如果容量都是整数，它一定会终止；如果随意选择路径，复杂度常写成 O(E * F)，其中 F 是最大流值。这个表达说明它可能受数值大小影响，而不只是受图规模影响。

延伸阅读：Ford-Fulkerson 推导用一个会选错初始路径的网络，展示残量网络和反向边如何修正已有流量。

2. Edmonds-Karp 算法

Edmonds-Karp 给 Ford-Fulkerson 加上固定策略：每次用 BFS 找边数最少的增广路径。这样路径选择不再随意，复杂度有稳定上界：

O(V * E^2)

它适合教学、验证和中小规模问题。实现直观，但在较大图上通常不如 Dinic 常用。

延伸阅读：Edmonds-Karp 推导重点解释 BFS 为什么能稳定选择最短增广路径，并逐轮计算最大流。

3. Dinic 算法

Dinic 的思想是“先分层，再批量增广”。它和 Hopcroft-Karp 的气质很接近：不再一条路一条路随便找，而是在层次图里尽可能找出一个阻塞流。

BFS 建层次图 -> 只保留向下一层的边 -> DFS 推阻塞流 -> 重新分层

当前弧优化会记录每个点已经尝试到哪条边，避免 DFS 反复扫描已证明无效的边。实践中，Dinic 是最大流问题里非常常用的实现选择。

延伸阅读：Dinic 推导从分层图、阻塞流和当前弧优化出发，展示它如何比逐条路径增广更高效。
算法找路方式一轮推进适合场景
Ford-Fulkerson任意增广路径一条路径理解最大流思想的基础框架。
Edmonds-KarpBFS 最短增广路径一条路径实现稳定，适合教学和小中规模验证。
DinicBFS 分层 + DFS一个阻塞流工程和竞赛中常用，适合更大规模的稀疏图。

六、常见建模套路

学最大流最有价值的部分，不是背某个模板，而是把业务约束翻译成节点、边和容量。

超级源点与超级汇点

多个供给点、多个需求点可以统一接到超级源点和超级汇点。

S -> 每个供给点，容量 = 供给量 每个需求点 -> T，容量 = 需求量

点容量转边容量

如果限制的是节点处理能力，可以把节点拆成入点和出点。

v_in -> v_out，容量 = 节点能力 入边连 v_in，出边从 v_out 发出

二分图匹配转最大流

一对一匹配可以看成所有容量为 1 的特殊最大流。

s -> 左侧点，容量 1 左侧点 -> 右侧点，容量 1 右侧点 -> t，容量 1

多对多分配

只要把容量从 1 改成对应上限，就能表达一对多或多对多。

仓库 -> 区域，容量 = 可配送能力 区域 -> t，容量 = 区域需求

用最大流判断可行性

如果总需求为 D，构图后求最大流：

max_flow == D 表示需求全部可满足 max_flow < D 表示存在供给、路径或容量瓶颈

更进一步，最小割还能告诉你瓶颈大致出在哪个边界上：是供给不足、转运链路不足，还是需求侧入口能力不足。

七、业务应用与边界

物流履约能力

工厂、仓库、分拨中心、配送站之间都有容量约束。最大流可以估算当前网络最多能履约多少订单，并辅助定位瓶颈。

仓配网络履约上限瓶颈识别

库存与需求分配

库存节点到区域需求节点的分配，可以用最大流判断满足量。如果再考虑成本、时效或收益，就会自然升级为最小费用流。

库存需求满足缺口诊断

服务容量评估

入口、服务集群和下游依赖可以抽象成网络，边容量表示 QPS 或处理上限。最大流给出系统吞吐的结构性上界。

服务架构QPS容量规划

匹配与派单

司机和订单、人员和任务、商品和展位的一对一或一对多关系，都可以用容量边表达可行匹配数量。

匹配派单资源分配

边界

最大流擅长回答“在容量约束下最多能满足多少”。它不直接处理复杂时间窗、非线性成本、动态反馈、多目标权衡、车辆路径顺序依赖等问题。这些通常需要最小费用流、整数规划、车辆路径算法、启发式搜索或仿真优化。

八、通向最小费用最大流

最大流只关心最多能送多少，但业务里经常还要问：在满足尽量多需求的前提下，怎样成本最低、时效最好或收益最高。

最大流：capacity 最小费用最大流：capacity + cost
问题边属性目标业务含义
二分图匹配可行关系，容量多为 1最大匹配数量最多能配成多少对。
最大流容量最大通过量最多能满足多少需求。
最小费用最大流容量 + 单位成本最大流量下成本最低既要满足需求，也要控制成本或收益。
最小费用最大流不是另起炉灶，而是在最大流的残量网络上继续引入费用、势能和最短路思想。
