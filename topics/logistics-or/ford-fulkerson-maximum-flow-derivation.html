一、Ford-Fulkerson 在解决什么

最大流问题给定一个有向网络，每条边有容量，从源点 s 到汇点 t，目标是找到能送到 t 的最大总流量。

目标：maximize |f| 容量约束：0 <= f(u, v) <= c(u, v) 流量守恒：除 s 和 t 外，每个中间点流入量 = 流出量
Ford-Fulkerson 的核心不是“找一条最好的路”，而是在一个动态变化的残量网络里，不断寻找还能让总流量增加的调整方案。

这句话里有两个关键词：增广路径和残量网络。前者表示还能让总流量增加的路径，后者表示当前流量状态下还能继续增加或撤销的空间。

二、从朴素推流开始

最自然的算法思路是：

找一条 s 到 t 的路径 -> 计算路径瓶颈容量 -> 沿整条路径送流 -> 重复直到没有路

如果路径 P 上的剩余容量分别是 5, 2, 4，那么这条路径一次最多只能增加：

delta = min(5, 2, 4) = 2

这个想法本身没错，问题出在“没有路”到底是什么意思。如果只在原图剩余容量里看路，早期路径选择可能让图看起来堵死；但如果允许撤销和重排历史流量，实际还可以继续增加。

三、一个会“选错路”的网络

看下面这个网络。源点有两条出边，总供给能力是 4 + 4 = 8；汇点有两条入边，总接收能力也是 4 + 4 = 8。真正的瓶颈不在源点或汇点，而在中间从左半区跨到右半区的两条边。

一眼看上去，最优方案应该是：

s -> A -> C -> t：送 2 s -> B -> D -> t：送 3 总流量 = 5

但是 Ford-Fulkerson 允许“任意找一条增广路径”。如果第一步路径选得不好，它会先走一条看似可行、但占用了中间关键边的路。

四、残量网络与反向边

假设第一轮选择了这条路径：

P1 = s -> A -> B -> D -> t delta = min(4, 3, 3, 4) = 3

推完以后，真实流量是：

s -> A: 3 / 4 A -> B: 3 / 3 B -> D: 3 / 3 D -> t: 3 / 4 当前总流量 |f| = 3

此时 A -> B 和 B -> D 都满了。如果只看原图正向剩余容量，就会觉得 B -> D 这条跨区通道已经被错误路径占掉，后续来自 s -> B 的供给很难再穿过中间瓶颈。

1. 残量网络记录两类能力

还能继续增加多少

原边 u -> v 还有 c - f 的正向残量，表示还能继续往前推多少。

最多能撤销多少

如果原边已经有 f 单位真实流量，残量网络中就有 v -> u 的反向残量 f。

正向残量：res(u, v) = c(u, v) - f(u, v) 反向残量：res(v, u) = f(u, v)

2. 第一轮之后的关键残量

第一轮之后，和后续调整有关的残量边包括：

残量边残量含义
s -> A1s -> A 还可以再送 1。
s -> B4s -> B 还完全没用。
B -> A3可以撤销原来 A -> B 上的 3 单位流。
A -> C2A -> C 还完全没用。
C -> t4C -> t 还完全没用。
反向边不是现实中的“货物倒流”，而是算法里的撤销权：沿 B -> A 走 1 单位，等价于把原来的 A -> B 真实流量减少 1。

五、用反向边完成重排

第一轮之后，继续找增广路径。先还可以走：

P2 = s -> A -> C -> t delta = min(1, 2, 4) = 1

推完以后，当前总流量变成 4。此时 s -> A 已满，但 s -> B 还有 4；真正的问题是：B -> D 这条跨区边已经被第一轮占满了。

关键的第三轮增广

残量网络中出现一条非常重要的路径：

P3 = s -> B -> A -> C -> t delta = min(4, 3, 1, 3) = 1

其中 B -> A 不是原图的真实边，而是 A -> B 的反向残量边。沿这条路径推 1，含义是：

路径边动作真实流量变化
s -> B走正向边s -> B 增加 1。
B -> A走反向边撤销 A -> B 上的 1。
A -> C走正向边A -> C 再增加 1。
C -> t走正向边C -> t 再增加 1。

最终真实流量变成：

s -> A: 4 / 4 s -> B: 1 / 4 A -> B: 2 / 3 A -> C: 2 / 2 B -> D: 3 / 3 C -> t: 2 / 4 D -> t: 3 / 4 总流量 |f| = 5
残量网络中的路径 s -> B -> A -> C -> t 不是最终真实业务路径。它是一套调整动作：让 B 接住来自 s 的新流量，同时把 A 原先错误送给 B 的流量撤回来，改送给 C。

六、最大流最小割证明过程

现在总流量已经达到 5。注意，这次不能再用“源点出边总容量”或“汇点入边总容量”直接解释，因为它们都是 8，明显大于当前流量。真正限制网络的是中间跨区边。

1. 先看终止时的残量网络

最终真实流量确定后，把每条边换算成正向残量和反向残量。和可达性有关的关键残量如下：

残量边残量说明
s -> A0s -> A 已满，不能从 s 直接走到 A。
s -> B3s -> B 还可走，所以 B 从 s 可达。
B -> A2这是 A -> B 的反向残量，所以 A 也从 s 可达。
A -> C0跨向右半区的上边已经满流。
B -> D0跨向右半区的下边也已经满流。

因此，残量网络中从 s 出发，最多只能到达 B 和 A，无法继续进入 C、D 或 t。

2. 从终止时的残量可达点构造割

最终状态下，残量网络中从 s 出发，仍然可以到达 B，再通过反向边 B -> A 到达 A。但从 A 到 C、从 B 到 D 都已经没有正向残量，所以无法进入右半区。

S = {s, A, B} T = {C, D, t}

从 S 指向 T 的原图边就是：

A -> C，容量 2 B -> D，容量 3 割容量 = 2 + 3 = 5

3. 为什么这就是最小割

任意从 s 到 t 的流量都必须穿过任何一个割，所以：

任意可行流 <= 任意割容量

现在我们找到一个可行流，流量是 5；同时找到一个割，容量也是 5。因此：

当前流量 5 <= 最大流 <= 最小割 <= 这个割容量 5 所以：最大流 = 最小割 = 5
Ford-Fulkerson 终止时，“没有增广路径”会自然给出一个割：残量网络里从 s 还能到达的点在一侧，其他点在另一侧。这个割的容量刚好等于当前流量。

七、算法流程与实现含义

Ford-Fulkerson 的完整流程可以写成：

初始化所有边流量为 0，并建立残量网络 while 残量网络中存在 s 到 t 的路径 P: delta = P 上所有残量边的最小残量 沿 P 增广 delta 更新每条边和反向边的残量 返回当前总流量

实现里通常只维护残量边。每添加一条原始边 u -> v，同时添加一条初始容量为 0 的反向边 v -> u。每次推流时统一更新：

edge.cap -= delta reverse_edge.cap += delta

这两行同时覆盖了“新增正向流”和“撤销历史流”。如果走的是正向残量边，就是增加真实流量；如果走的是反向残量边，就是减少对应原边上的真实流量。

走到的边更新后意味着什么业务解释
原图正向边正向残量减少，反向残量增加这条通道被更多使用，同时获得未来撤销权。
反向残量边反向残量减少，原正向残量增加撤销之前占用的容量，把它让给更合适的组合。

八、复杂度与学习边界

Ford-Fulkerson 是方法框架，不是唯一实现。复杂度取决于每次怎么找增广路径。

版本找路径方式复杂度直觉特点
朴素 Ford-FulkersonDFS 或任意路径O(E * F)容易理解，但路径选择差时可能很慢。
Edmonds-KarpBFS 找边数最少的增广路径O(V * E^2)选择规则稳定，适合教学和验证。
DinicBFS 分层 + DFS 推阻塞流实践表现通常更好工程和竞赛中更常用。
理论边界

如果容量不是整数，朴素 Ford-Fulkerson 在理论上可能出现不收敛问题。多数业务建模中的订单数、库存数、车辆数、QPS 配额都可以看作整数或离散容量；工程实现中也常选择 Edmonds-Karp 或 Dinic 来获得更稳定的行为。
