Dijkstra + 势能优化：从 reduced cost 理解最小成本最大流

SPFA 版本把最短路增广的原理讲得很清楚，但工程上更常用 Dijkstra + 势能。势能的作用不是改变原问题，而是把残量网络里的负费用边重标定成非负边权，让 Dijkstra 可以稳定地找到最短费用增广路。

一、为什么需要势能

最小成本最大流的入门算法通常是：在残量网络中找一条费用最小的 s -> t 路径，沿路径增广，然后更新正向边和反向边。问题在于，残量网络里会出现负费用反向边。

原边 u -> v 的单位费用是 c 如果已经有 1 单位流走过 u -> v 残量网络中会出现 v -> u，单位费用是 -c

SPFA 能处理负边权，所以它适合入门；Dijkstra 默认要求边权非负，所以不能直接在带负反向边的残量网络上跑。

势能要解决的核心问题是：不改变哪条路径最短，却把每条可用残量边的“计算费用”变成非负数。

这个变换后的费用通常叫 reduced cost，也可以翻译成“约化费用”或“重标定费用”。

二、核心概念

1. 势能 potential

给每个点一个数，记为 potential[v]。它可以理解为节点的价格、海拔或对偶变量。算法不会直接拿原始费用 cost(u, v) 跑 Dijkstra，而是使用约化费用：

reduced_cost(u, v) = cost(u, v) + potential[u] - potential[v]

只要对每条有残量的边都满足：

reduced_cost(u, v) >= 0

Dijkstra 就可以安全运行。

2. 为什么路径最短关系不变

对任意一条从 s 到 t 的路径，所有中间点的势能会一正一负抵消：

sum(reduced_cost) = sum(cost(u, v) + potential[u] - potential[v]) = sum(cost) + potential[s] - potential[t]

对于同一轮搜索，potential[s] - potential[t] 是常数，所以路径之间的相对大小不变。

Dijkstra 在约化费用上找到的最短路，仍然对应真实费用意义下的最短增广路。势能只是换了一把尺子，没有换问题。

3. 初始势能怎么来

场景初始化方式说明
原始费用非负potential[v] = 0初始残量图只有原边，没有负费用反向边，可以直接跑第一轮 Dijkstra。
存在负费用原边先跑一次 Bellman-Ford / SPFA用从 s 到各点的最短距离作为初始势能，使约化费用非负。
存在负费用环不能直接套普通最短路增广需要先处理负环或换成更完整的最小费用循环 / 原始对偶框架。

4. 每轮如何更新势能

Dijkstra 结束后，得到当前约化费用下从 s 到每个点的最短距离 dist[v]。对可达点更新：

potential[v] = potential[v] + dist[v]

这一步把刚刚发现的最短路信息吸收到节点价格中。更新之后，下一轮残量图里的可用边仍然可以保持非负约化费用。

三、算法流程

Dijkstra + 势能版最小成本最大流仍然是“最短路增广”。它和 SPFA 版本的不同点只有一个：每轮最短路用约化费用，而不是直接用真实费用。

初始化势能 -> 用 reduced cost 跑 Dijkstra -> 恢复真实路径费用 -> 更新势能 -> 沿路径增广 
flow = 0
total_cost = 0
potential[v] = 0  // if all original costs are non-negative

while t is reachable by Dijkstra on reduced_cost:
    path_cost = dist[t] + potential[t] - potential[s]

    for each reachable vertex v:
        potential[v] += dist[v]

    delta = min residual capacity on the restored s -> t path

    for each edge e on the path:
        e.cap -= delta
        e.rev.cap += delta

    flow += delta
    total_cost += delta * path_cost
顺序说明

实际代码里常见写法是先根据 dist 更新势能，再沿最短路增广；也可以先记录真实路径费用，再更新势能。关键是不要把约化费用 dist[t] 直接当成真实费用。

四、示例建模

继续使用一个 2 个工人、2 个任务的最小成本指派例子。每个工人最多做 1 个任务，每个任务需要 1 个工人。中间边上的数字是单位成本。

工人任务 X任务 Y
A12
B2100

直觉上，第一轮最便宜路径会选择 A -> X。但要完成两个任务，最终最优方案是 A -> Y 和 B -> X，总成本是 4。第二轮必须经过反向边 X -> A 撤销第一轮的选择。

五、逐轮推导

第 0 步：初始势能

所有原始费用非负，所以初始势能全部设为 0：

potential[s] = potential[A] = potential[B] = potential[X] = potential[Y] = potential[t] = 0

此时约化费用等于真实费用，Dijkstra 可以直接运行。

第 1 轮：走最便宜的 A -> X

点Dijkstra 距离 dist解释
s0源点。
A0s -> A 成本 0。
B0s -> B 成本 0。
X1通过 A -> X。
Y2通过 A -> Y。
t1s -> A -> X -> t。

第一条增广路径是：

s -> A -> X -> t 约化费用 = 1 真实费用 = dist[t] + potential[t] - potential[s] = 1 + 0 - 0 = 1

沿路径增广 1 单位流。然后把本轮最短距离吸收到势能：

potential[s] = 0 potential[A] = 0 potential[B] = 0 potential[X] = 1 potential[Y] = 2 potential[t] = 1

第 1 轮之后：负费用反向边被势能压平

因为 A -> X 已经走了 1 单位流，残量网络中出现反向边 X -> A，真实费用是 -1。现在看它的约化费用：

reduced_cost(X, A) = cost(X, A) + potential[X] - potential[A] = -1 + 1 - 0 = 0

这就是势能的核心效果：真实费用中出现了负边，但 Dijkstra 看到的是非负约化费用。

第 2 轮：用反向边重排第一轮选择

第二轮在约化费用上跑 Dijkstra，关键边如下：

残量边真实费用约化费用含义
s -> B00B 还没被使用。
B -> X21B 可以接任务 X。
X -> A-10撤销第一轮的 A -> X。
A -> Y20让 A 改接任务 Y。
Y -> t01任务 Y 被完成。

最短增广路径是：

s -> B -> X -> A -> Y -> t 约化费用 = 0 + 1 + 0 + 0 + 1 = 2 真实费用 = dist[t] + potential[t] - potential[s] = 2 + 1 - 0 = 3

这 3 的含义很重要：它不是最终总费用，而是第二轮在当前流基础上的边际调整费用。

当前总费用 = 1 第二轮边际费用 = B->X(2) + X->A(-1) + A->Y(2) = 3 最终总费用 = 1 + 3 = 4

沿第二条路径增广后，X -> A 会撤销 A -> X 的 1 单位流，最终方案变成：

A -> Y

成本 2

A 不再做 X，而是做 Y。

B -> X

成本 2

B 接手 X，避开了 B -> Y 的高成本。

最终最大流量是 2，总费用是 4。势能没有阻止反向边发挥作用，只是把负费用反向边换算成 Dijkstra 可以处理的非负约化费用。

六、正确性直觉

1. 约化费用不改变最短路

前面已经看到，对任意 s -> t 路径：

reduced_path_cost = real_path_cost + potential[s] - potential[t]

同一轮里，起点和终点固定，所以所有路径都被加上同一个常数。谁真实费用最小，谁的约化费用也最小。

2. 势能更新保持非负边权

假设当前所有可用残量边都有非负约化费用。Dijkstra 得到距离 dist 后，对于任意从可达点 u 指向 v 的残量边，都有最短路松弛条件：

dist[v] <= dist[u] + reduced_cost(u, v)

更新势能后，新约化费用是：

new_reduced_cost(u, v) = cost(u, v) + new_potential[u] - new_potential[v] = old_reduced_cost(u, v) + dist[u] - dist[v] >= 0

因此下一轮仍然能用 Dijkstra。沿最短路增广后，新出现的反向边也对应这条最短路上的等式边，约化费用为 0，不会破坏非负性。

3. 为什么仍然是最小费用增广

每一轮 Dijkstra 找到的是当前残量网络中的最短费用增广路径。残量网络同时包含新增和撤销两类动作，所以它不是在原始图中做简单贪心，而是在当前流方案附近找最低成本的可行调整。

从 SPFA 到 Dijkstra + 势能，最优性来源没有改变：仍然是残量网络中的最短费用增广。势能只是让最短路这一步更稳定。
证明边界

严格证明通常会放在原始对偶算法框架里，用可行势能、互补松弛和最短路最优性来说明。本文的关键是先掌握三个可手推的不变式：路径排序不变、约化费用非负、每轮仍是最短费用增广。

七、手写实现要点

1. 边结构仍然要记录反向边

Dijkstra + 势能只替换了最短路算法，残量网络结构和 SPFA 版本完全一样。

struct Edge {
    int to;
    int rev;
    int cap;
    int cost;
};

2. Dijkstra 松弛的是约化费用

松弛时不要直接使用 edge.cost，而要使用：

rcost = edge.cost + potential[u] - potential[v]
if edge.cap > 0:
    rcost = edge.cost + potential[u] - potential[edge.to]
    if dist[edge.to] > dist[u] + rcost:
        dist[edge.to] = dist[u] + rcost
        parent_node[edge.to] = u
        parent_edge[edge.to] = edge_index

3. 总费用要用真实费用

dist[t] 是约化费用距离，不是真实路径费用。更新总费用时使用：

real_path_cost = dist[t] + potential[t] - potential[s] total_cost += delta * real_path_cost

4. 势能只更新可达点

如果某个点本轮不可达，dist[v] 是无穷大，不应该拿来更新势能。

for each vertex v:
    if dist[v] < INF:
        potential[v] += dist[v]

5. 推荐对拍顺序

手写时不要一上来就只测大样例。先让 SPFA 版本和 Dijkstra + 势能版本在小图上互相对拍，再上随机费用流。
