LOGISTICS OR KNOWLEDGE MAP

物流运筹算法核心知识地图

这份地图聚焦物流场景中的核心运筹知识：建模语言、问题类型、算法工具、场景对应、评估与工程化落地。已补充常见遗漏模块，包括约束规划、排队论、随机/鲁棒优化、分解算法、仿真与调度。

物流运筹的核心不是背公式，而是把真实业务问题翻译成：决策变量、目标函数、约束条件、输入数据、输出结果和评估指标。

一、总览：从业务问题到算法系统

物流系统里的算法决策通常沿着履约链路展开。每一个环节都可以抽象成不同类型的优化问题。

需求预测→ 库存分配→ 订单分仓→ 仓内作业→ 干线调拨→ 末端配送→ 异常恢复
物流环节典型决策对应运筹问题
需求预测未来订单量、品类需求、区域需求如何变化预测模型 + 随机优化 / 鲁棒优化
库存分配货放在哪、放多少、何时补库存优化、随机规划、多级库存控制
订单分仓订单从哪个仓发，是否拆合单指派问题、整数规划、组合优化
仓内作业波次划分、拣货路径、人员排班调度问题、TSP、排队论、离散事件仿真
干线调拨货在仓、城市、分拨之间如何流动网络流、最小费用流、多商品流、网络设计
末端配送车辆怎么走、订单怎么派TSP、VRP、VRPTW、动态优化
异常恢复爆仓、延误、缺货后如何重算在线优化、滚动优化、鲁棒优化、重调度

二、第一层：建模语言

建模能力是物流运筹的入口。零基础学习时，最先要掌握的不是求解算法，而是如何把业务问题翻译成优化模型。

决策变量

我要决定什么？

例如订单 i 是否由仓 j 发货，车辆 k 是否访问配送点 p，每条线路运输多少件货。

目标函数

我要优化什么？

例如总成本最低、总距离最短、准时率最高、拆单最少、仓库负载更均衡。

约束条件

哪些规则不能违反？

例如库存不能超、车辆容量不能超、订单必须准时、司机工作时长有限、线路有容量。

评估指标

怎么证明策略更好？

例如成本、时效、超时率、无解率、fallback 触发率、求解耗时、现场可执行率。

通用建模模板

业务问题：要解决哪个物流决策？
决策变量：系统需要决定什么？
目标函数：希望优化哪个指标？
硬约束：绝对不能违反的条件是什么？
软约束：尽量满足但可以付出惩罚的偏好是什么？
输入数据：订单、库存、车辆、线路、时效、成本等
输出结果：分仓方案、路径方案、调拨方案、排班方案等
异常情况：无解、超时、数据异常、约束冲突时怎么处理？
评估指标：用什么指标判断方案好坏？

三、第二层：核心问题类型

指派问题

分给谁

用于订单分仓、司机派单、任务分配。核心是把一组任务分配给一组资源。

  • 成本矩阵
  • 二分图匹配
  • 匈牙利算法
  • 容量约束指派
路径问题

怎么走

用于末端配送、干线路径、仓内拣货。核心是选择访问顺序和线路。

  • 最短路径
  • TSP
  • VRP
  • VRPTW
网络流问题

怎么流

用于干线调拨、多仓供给、城市需求满足。核心是在有容量和成本的网络上安排货物流动。

  • 最大流
  • 最小费用流
  • 多商品流
  • 网络设计
数学规划

选不选、派不派

用于复杂约束下的选择与组合决策。许多物流问题最终都会被表达成 LP、IP、MIP 或 CP 模型。

  • LP / IP / MIP
  • CP-SAT / 约束规划
  • 0-1 变量
  • 硬约束与软约束
库存优化

放多少

用于备货、补货、安全库存、多仓库存配置。重点是用库存成本换服务水平。

  • 安全库存
  • 补货点
  • 缺货成本
  • 需求不确定性
调度问题

什么时候做

用于司机排班、仓内波次、分拣线任务、干线班车。重点是资源、时间和任务之间的匹配。

  • Scheduling
  • Job Shop / Flow Shop
  • 时间窗
  • 作业瓶颈
选址与覆盖

建在哪里、覆盖谁

用于仓网规划、前置仓布局、站点服务范围划分。

  • Facility Location
  • Set Covering
  • Max Coverage
  • p-median / p-center
随机与鲁棒优化

不确定时怎么决策

用于需求波动、时效波动、大促峰值、天气和异常扰动。

  • Stochastic Programming
  • Robust Optimization
  • Chance Constraints
  • Scenario Planning
仿真与排队

系统会不会堵

用于仓内瓶颈、分拨能力、站点拥堵、大促压测和策略回放。

  • Queueing Theory
  • Discrete Event Simulation
  • Monte Carlo
  • What-if 分析

六、第五层：评估与工程化落地

工业场景下，运筹算法不是“求出一个答案”就结束。它必须能被评估、上线、监控、解释和回滚。

效果指标
  • 履约成本
  • 平均时效 / 准时率
  • 超时率 / 违约率
  • 拆单率
  • 仓库负载均衡
  • 车辆利用率
系统指标
  • 求解耗时
  • 无解率
  • 超时率
  • fallback 触发率
  • 数据异常率
  • 任务恢复成功率
评估方法
  • 历史订单回放
  • 策略 A/B 对比
  • What-if 分析
  • 极端场景压测
  • 离散事件仿真
  • 线上灰度实验
工程化能力
  • 标准输入输出
  • 约束配置
  • 异步求解任务
  • 状态流转
  • 超时与降级
  • 结果解释与归因
