SPFA 最短路增广：从残量网络理解最小成本最大流

最小成本最大流可以先从“每次找当前最便宜的增广路径”学起。SPFA 版本不一定是最快的工程实现，但它把容量、费用、残量网络、反向边和重排历史流量这些核心概念暴露得最清楚。

一、从哪里开始理解

最大流只关心“最多能送多少流量”。最小成本最大流多问一步：在最大流量已经确定的前提下，哪一种流量安排的总费用最低。

最大流：maximize flow 最小成本最大流：先 maximize flow，再 minimize sum(flow(e) * cost(e))
SPFA 最短路增广的入口直觉是：每增加一批流量，都选择当前残量网络里单位边际成本最低的 s -> t 路径。

这里有两个重要限定。第一，最短路不是边数最少，而是路径费用和最小。第二，路径不是在原始图里找，而是在残量网络里找；残量网络中会出现负费用反向边，所以入门实现常用能处理负边权的 SPFA。

为什么适合入门

SPFA 代码结构接近普通最短路，容易看到每轮如何选路径、算瓶颈、更新正反边。

为什么不是终点

SPFA 的速度受数据影响很大。大规模工程实现通常会转向 Dijkstra + 势能优化。

二、核心概念

最小成本最大流的每条边至少包含三类信息：容量、费用、当前流量。

概念符号直觉
容量cap(u, v)这条边最多允许通过多少流量。
费用cost(u, v)每通过 1 单位流量，要付出的单位成本。
流量flow(u, v)当前已经安排在这条边上的流量。
残量cap - flow这条正向边还可以继续增加多少流量。

1. 正向边与反向边

每加入一条真实边 u -> v，算法实现里通常同时加入一条反向边 v -> u。反向边刚开始容量为 0；当正向边走了流量，反向边就拥有同等的可撤销容量。

真实边 u -> v: residual_capacity = cap - flow residual_cost = cost 反向边 v -> u: residual_capacity = flow residual_cost = -cost
反向边不是业务里的反向运输，而是算法里的“撤销按钮”。走一条费用为 -c 的反向边，表示撤销此前在原边上花过的 c。

2. 路径费用与瓶颈流量

残量网络中的一条 s -> t 路径，代表一种可以让总流量继续增加的调整方案。路径费用是这次增加 1 单位流量的边际成本；瓶颈流量是这条路径一次最多能增广多少。

path_cost = sum(cost(e)), e 属于路径 delta = min(residual_capacity(e)), e 属于路径 total_cost += delta * path_cost

三、SPFA 在做什么

SPFA 可以理解为 Bellman-Ford 的队列优化版本。它维护每个点从源点 s 出发的当前最小费用距离 dist，只要某条边能让终点距离变小，就更新距离和父指针，并把终点重新放入队列。

如果 residual_capacity(u, v) > 0 并且 dist[v] > dist[u] + cost(u, v) 则更新： dist[v] = dist[u] + cost(u, v) parent[v] = u -> v

费用流里使用 SPFA 的关键原因是：残量网络中会有负费用反向边。普通 Dijkstra 不能直接处理负边权，而 SPFA 可以在没有负费用环干扰时找到最短费用路径。

边界条件

如果初始图或残量网络存在可利用的负费用环，最短路会失去稳定定义，需要先做负环处理，或使用更完整的最小费用循环 / 原始对偶框架。常见的指派、运输、分仓模型通常使用非负原始费用，负边主要来自反向撤销边。

四、最短路增广流程

SPFA 最短路增广版的最小成本最大流可以写成一个很短的循环：

初始化流量为 0 -> 在残量网络跑 SPFA -> 还原最短费用路径 -> 按瓶颈增广 -> 重复直到 t 不可达 
flow = 0
cost = 0

while SPFA can reach t in the residual graph:
    path = restore by parent pointers
    delta = min residual capacity on path

    for each residual edge e on path:
        e.capacity -= delta
        e.reverse.capacity += delta

    flow += delta
    cost += delta * dist[t]

return flow, cost

注意最后的 dist[t] 是当前残量网络中从 s 到 t 的最小路径费用，也就是这轮每增加 1 单位流量的边际成本。

五、示例建模

下面用一个 2 个工人、2 个任务的例子推导。每个工人最多做 1 个任务，每个任务需要 1 个工人。边上的标签写成 容量 / 费用。

工人任务 X任务 Y
A12
B2100

单看第一条边，A -> X 成本最低，算法第一轮会选它。但如果最终要做满两个任务，最优方案其实是 A -> Y 加 B -> X，总成本 2 + 2 = 4。这个例子正好能演示反向边如何撤销第一轮选择。

六、逐轮推导

第 1 轮：选当前最便宜路径

初始时所有流量为 0。SPFA 从 s 出发，能得到下面这些候选路径费用：

路径费用是否最短
s -> A -> X -> t0 + 1 + 0 = 1是
s -> A -> Y -> t0 + 2 + 0 = 2否
s -> B -> X -> t0 + 2 + 0 = 2否
s -> B -> Y -> t0 + 100 + 0 = 100否
P1 = s -> A -> X -> t delta = min(1, 1, 1) = 1 flow = 1 total_cost = 1

增广后，s -> A、A -> X、X -> t 都被占满。残量网络里会出现三条反向边：A -> s 费用 0、X -> A 费用 -1、t -> X 费用 0。

为什么第一轮后还要继续？找到 P1 只是找到了“一种可以增加 1 单位流量的办法”。增广是把这件事真正写入当前流量：把 s -> A、A -> X、X -> t 的真实流量都加 1。此时总流量只有 1，但网络里还存在从 s 到 t 的残量路径，所以最小成本最大流还不能停。最大流目标下的继续条件是：SPFA 还能在残量网络中到达 t；如果是指定流量 K 的最小费用流，则是还没送满 K 且 t 仍可达。

第 2 轮：SPFA 使用反向边重排

如果没有反向边，下一条看似可行的直接路径只有 s -> B -> Y -> t，费用高达 100。但残量网络里还有一条更便宜的调整路径：

P2 = s -> B -> X -> A -> Y -> t path_cost = 0 + 2 + (-1) + 2 + 0 = 3 delta = 1

这条路径里的 X -> A 是反向边，含义是撤销第一轮的 A -> X。增广后，第一轮的安排被重排为：A 改去做 Y，B 改去做 X。

第 2 轮不是新增一条最终分配路径，而是一组调整动作： s -> B: 增加 1 B -> X: 增加 1 X -> A: 走反向边，所以 A -> X 减少 1 A -> Y: 增加 1 Y -> t: 增加 1

第 2 轮更新后的真实流量

沿 P2 增广 1 单位后，真实流量变化如下：

边变化解释
s -> B增加 1B 开始接一个任务。
B -> X增加 1B 接下任务 X。
X -> A走反向边撤销原来的 A -> X。
A -> Y增加 1A 改去做任务 Y。
Y -> t增加 1任务 Y 被完成。

A -> X 和 X -> A 的“抵消”，只发生在真实边 A -> X 的流量上：第一轮把 A -> X 加到 1，第二轮走反向边 X -> A 又把它减回 0。注意 X -> t 没有被撤销，它保留第一轮的 1 单位流量；第二轮新增的 B -> X 正好接上这条出口，所以任务 X 最终由 B 完成。

flow = 2 total_cost = 第 1 轮成本 1 + 第 2 轮边际成本 3 = 4 最终分配：A -> Y，B -> X

终止

最终 s -> A 和 s -> B 都已经满流，源点无法再到达任何有正向供给残量的工人，因此残量网络中不存在新的 s -> t 路径。算法终止，当前流量 2 是最大流；在流量为 2 的所有方案里，总费用 4 是最小的。

这个例子的重点不是算出 4，而是看见“反向边 + 负费用”如何让算法改正第一轮局部选择：先走 A -> X 没问题，因为第二轮可以通过 X -> A 把这 1 单位流量退回来，再交给更合适的组合。

七、为什么能得到低成本最大流

最短路增广有一个关键不变式：每一轮都在当前残量网络中选择边际成本最低的可增广方案。残量网络同时包含“继续增加”和“撤销重排”两种动作，所以它看到的不是局部贪心的静态选择，而是对当前整套流量方案的最便宜调整。

容量正确性

每轮只沿残量容量大于 0 的边增广，且按路径瓶颈推流，所以不会违反容量约束。

费用改进性

反向边允许撤销历史流量。只要有更低成本的重排机会，它会表现为残量网络中的便宜路径或负费用调整。

当残量网络里已经没有 s -> t 路径时，说明无法继续增加总流量，当前流是最大流。由于每个流量层级都是通过最短费用增广路径推进，并持续保留可撤销调整，最终得到的是最大流量下的最小费用方案。

不要误解

它不是“每次选原始图中最便宜边”。它每次选的是残量网络里的最短费用路径，路径中可能混合正向新增边和反向撤销边，这才是费用流区别于简单贪心分配的核心。

八、实现要点

1. 边结构必须记录反向边下标

实现时，一条真实边和一条反向边要互相知道对方在邻接表里的位置。这样沿路径增广时，才能同时减少当前边残量、增加反向边残量。

add_edge(u, v, cap, cost):
    forward = Edge(to=v, rev=len(graph[v]), cap=cap, cost=cost)
    backward = Edge(to=u, rev=len(graph[u]), cap=0, cost=-cost)
    graph[u].append(forward)
    graph[v].append(backward)

2. SPFA 只松弛残量大于 0 的边

已经满流的正向边不能继续走；没有可撤销流量的反向边也不能走。判断条件通常写成：

if edge.cap > 0 and dist[edge.to] > dist[u] + edge.cost: relax

3. 父指针记录的是边，不只是点

费用流更新时必须知道路径上到底用了哪条边，因为同一对点之间可能有多条边，且需要通过边的反向下标更新残量。

parent_node[v] = u parent_edge_index[v] = i // graph[u][i] 是这次最短路使用的边

4. 复杂度直觉

如果每次至少增广 1 单位整数流，最多增广 F 轮。SPFA 单轮常见粗略上界按 O(VE) 理解，因此教学版常写成：

O(F * V * E)

这个表达更像风险提示：当最大流量数值很大，或者图结构让 SPFA 反复入队时，它会变慢。理解清楚 SPFA 版本之后，再学习 Dijkstra + 势能优化会自然很多。

最小成本最大流的核心不是 SPFA 本身，而是“在残量网络中用最短费用路径做可撤销的增广”。SPFA 只是最直观的第一把钥匙。
