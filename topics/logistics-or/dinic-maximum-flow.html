Dinic 算法：分层图、阻塞流与最大流推导

Dinic 是最大流问题中非常常用的算法。它延续 Ford-Fulkerson 的残量网络思想，但不再一条增广路径一条增广路径地慢慢找，而是每轮先用 BFS 构造分层图，再用 DFS 在分层图中尽量推到一个阻塞流。

一、问题与前置模型

最大流问题给定一个有向容量网络 G = (V, E)，每条边 u -> v 有容量 c(u, v)，要求在源点 s 到汇点 t 之间安排最大可行流。

容量约束：0 <= f(u, v) <= c(u, v) 流量守恒：除 s、t 外，每个点的流入 = 流出 最大流值：|f| = 从 s 流出的总量 = 流入 t 的总量

Dinic 仍然工作在残量网络上。对每条原边 u -> v，残量网络同时表达两件事：

残量边残量容量含义
正向残量边 u -> vc(u, v) - f(u, v)还可以继续往这条原边增加多少流。
反向残量边 v -> uf(u, v)最多可以撤销多少已经安排在原边上的流。
Dinic 的改进点不在于换了最大流模型，而在于每一轮先建立“最短距离方向”，再在这个方向里批量增广。

二、Dinic 的核心思想

朴素 Ford-Fulkerson 每次只找一条增广路径；Edmonds-Karp 每次用 BFS 找一条边数最少的增广路径；Dinic 则进一步说：既然 BFS 已经得到了从 s 出发的层次关系，那就不要只用其中一条路，而要在这张分层图里一次性尽量多推。

BFS 分层 -> 保留向下一层的边 -> DFS 连续增广 -> 得到阻塞流 -> 重建分层图
一句话：Dinic 每轮不是找“一条最短增广路”，而是在当前最短层次结构里找“一批互不容易继续扩展的增广路”。
只增广一条路径

每次路径选择带来的收益有限，很多边会被反复扫描，尤其在多条短路径并存时显得浪费。

推到阻塞流

一轮内尽量把当前层次图里的可用通道推满，让下一轮必须寻找更深的增广结构。

三、核心概念

1. 层次与分层图

在残量网络中，从 s 出发做 BFS。若某点第一次被访问时距离为 level[x]，表示它从 s 至少需要经过多少条残量边才能到达。

level[s] = 0 如果残量边 u -> v 可走，且 v 未访问： level[v] = level[u] + 1

分层图只保留满足下列条件的残量边：

residual(u, v) > 0 level[v] = level[u] + 1

也就是说，DFS 只能沿着层数严格增加的方向走。这样可以避免在一轮内绕圈，也保证当前轮处理的是最短增广路径所在的层次结构。原图中的边不必然属于当前分层图；如果一条边没有从第 k 层指向第 k + 1 层，它这一轮就不能被 DFS 使用。

2. 阻塞流

在当前分层图里，如果已经不存在从 s 到 t 的可行路径，就称这一轮得到了一个阻塞流。

阻塞流不一定是全局最大流；它只是把当前这张分层图里的所有 s 到 t 通道都堵住了。

堵住的原因可能是源点侧的某条边满了，也可能是中间瓶颈满了，还可能是某条边在这一轮不符合分层方向。关键是：这一轮内再沿当前层次方向已经无法继续到达 t。

3. 当前弧优化

DFS 从一个点出发时，会依次尝试它的出边。如果某条边已经证明走不通，下一次再从这个点搜索时就没有必要从头扫描。当前弧优化用 it[u] 记录点 u 已经尝试到第几条边。

没有当前弧：每次 DFS 都可能重复扫描失败边 有当前弧：失败边只会被跳过一次，后续从记录位置继续

当前弧只在同一张分层图内有效。下一轮 BFS 会重建层次，原来不合法的边可能变合法，所以 it[u] 必须重置。

四、算法流程

Dinic 的主循环非常紧凑：

max_flow = 0
while BFS builds levels and t is reachable:
    reset current arc pointer it[u] = 0
    while pushed = DFS(s, infinity):
        max_flow += pushed
return max_flow

其中 DFS 的含义是：从当前点 u 出发，尝试沿分层图把不超过 limit 的流量送到 t。

DFS(u, limit):
    if u == t:
        return limit
    for edge i from it[u] to end:
        it[u] = i
        if residual(edge) > 0 and level[v] == level[u] + 1:
            pushed = DFS(v, min(limit, residual(edge)))
            if pushed > 0:
                apply pushed on edge and reverse edge
                return pushed
    return 0
很多模板里的 DFS 一次只返回一条路径的推流量；外层 while DFS(s, inf) 会反复调用，直到当前分层图中再也推不出流量。也有写法在一次 DFS 中累积多条分支，本质相同。

五、逐轮推导

考虑如下容量网络。网络包含横向边 A -> B，但不存在 B -> C。第一轮中 A -> B 因为 A 和 B 同层而不能进入分层图；当 s -> B 被堵住后，第二轮 B 的层数后移，A -> B 变成合法的分层边。

第 1 轮 BFS：构造分层图

初始残量等于容量，因此 BFS 得到：

level[s] = 0 level[A] = level[B] = 1 level[C] = level[D] = 2 level[t] = 3

分层图保留 s -> A、s -> B、A -> C、B -> D、C -> t、D -> t。它不保留 A -> B，因为此时 A 和 B 都在第 1 层，A -> B 不是从第 k 层走到第 k + 1 层。

第 1 轮 DFS：在分层图里推阻塞流

假设邻接扫描顺序为 s: A, B，A: C, B，B: D。DFS 会连续推进两条当前分层图中的路径：

路径瓶颈推完后的关键变化累计流量
s -> A -> C -> tmin(10, 4, 4) = 4A -> C 和 C -> t 满4
s -> B -> D -> tmin(3, 7, 7) = 3s -> B 满，B -> D 和 D -> t 各剩 47

此时当前分层图已经被堵住：s -> B 满了，A -> C 满了，而 A -> B 仍然不是这一轮的合法分层边。虽然 B -> D 和 D -> t 还各剩 4，当前分层图中已经没有从 s 到 t 的路径；阻塞流只表示当前分层图被堵住，不表示已经达到全局最大流。

当前主要流量： s -> A = 4 / 10 s -> B = 3 / 3 A -> C = 4 / 4 A -> B = 0 / 4 B -> D = 3 / 7 C -> t = 4 / 4 D -> t = 3 / 7

当前弧在第 1 轮如何起作用

当前弧优化不改变搜索结果，只减少重复扫描。第 1 轮里，A -> C 被推满后，再从 A 搜索时会发现 A -> C 不能走；接着检查 A -> B，又发现它不是下一层边。于是 it[A] 移动到末尾，后续同一轮不再从 A -> C 重新扫起。

时刻it[s]it[A]发生了什么
DFS 第一次进入s -> AA -> C成功推 s -> A -> C -> t，推入 4。
再次尝试 As -> AA -> C 后移A -> C 已满，A -> B 是同层边，A 在本轮不再产生可行分支。
继续从 s 尝试s -> B本轮末尾转去 s -> B -> D -> t，推入 3。
当前弧的作用在于：同一张分层图里，如果从 A 出发已经证明所有可选边都失败，后续 DFS 不会重复扫描这些失败边。

第 2 轮 BFS：层次发生变化

第 1 轮结束后重新 BFS。由于 s -> B 已满，B 不再是第 1 层；而 s -> A 还剩 6，A -> B 还剩 4，因此 B 会被 A -> B 带到第 2 层。

level[s] = 0 level[A] = 1 level[B] = 2 level[D] = 3 level[t] = 4

同一条 A -> B 在不同轮次中的角色发生了变化：第 1 轮因为 A 和 B 同层而被排除；第 2 轮因为 B 后移到第 2 层，成为继续增广的关键通道。

第 1 轮：A(L1) -> B(L1)，不是分层边 第 2 轮：A(L1) -> B(L2)，成为分层边

这一轮 DFS 找到：

P = s -> A -> B -> D -> t delta = min(6, 4, 4, 4) = 4

推完以后：

边流量 / 容量正向残量反向残量
s -> A8 / 1028
A -> B4 / 404
B -> D7 / 707
D -> t7 / 707

当前弧为什么要在第 2 轮重置

第 1 轮里 A -> B 被当前弧跳过，是因为它当时不符合分层方向；第 2 轮重新 BFS 后，A -> B 已经变成合法边。如果 it[A] 不重置，算法就会错过这条新合法路径。

每轮 BFS 后： level[] 重新计算 it[u] = 0 重新开始扫描 原因：边是否合法，取决于本轮 level[]，不是永久属性。

第 3 轮 BFS：终止并给出割

第 2 轮推完后，当前总流量为：

第一轮阻塞流：4 + 3 = 7 第二轮阻塞流：4 总流量：11

再次 BFS 时，s -> A 还剩 2，但 A -> C 和 A -> B 都已经满了；s -> B 也已经满了。所以残量网络中从 s 只能到达 A，无法再到达 t。

S = {s, A} T = {B, C, D, t} 跨割边： s -> B，容量 3 A -> C，容量 4 A -> B，容量 4 割容量 = 3 + 4 + 4 = 11

当前流量等于这个割的容量，所以最大流就是 11。

最大流对应的路径组合

最大流算法最终维护的是每条边上的流量，不一定直接输出“路径清单”。但在这个例子里，可以把最终边流量分解成三条从 s 到 t 的路径：

路径流量来源
s -> A -> C -> t4第 1 轮阻塞流中的第一条路径。
s -> B -> D -> t3第 1 轮阻塞流中的第二条路径。
s -> A -> B -> D -> t4第 2 轮重新分层后出现的更长路径。

把这三条路径叠加起来，就得到最终的边流量：

s -> A = 4 + 4 = 8 s -> B = 3 A -> C = 4 A -> B = 4 B -> D = 3 + 4 = 7 C -> t = 4 D -> t = 3 + 4 = 7 总流量 = 4 + 3 + 4 = 11

其中 B -> D 和 D -> t 是两条路径共享的后半段：来自 s -> B -> D -> t 的 3 单位流量，与来自 s -> A -> B -> D -> t 的 4 单位流量在 B 之后合并，所以这两条边上的最终流量是 7。

一般最大流结果也可以做路径分解：在所有正流量边组成的图中，反复找一条 s -> t 路径，取路径上的最小流量作为该路径流量并从沿途边上扣除，直到不存在正流量的 s -> t 路径。若存在环流，它不影响最大流值，可以单独忽略或消去。
该网络展示了 Dinic 的层次推进：第一轮只处理 3 条边的最短路径；第二轮允许 A -> B 进入分层图，处理 4 条边的更长路径。

六、为什么正确

Dinic 的正确性仍然来自 Ford-Fulkerson 的增广框架和最大流最小割定理。它每次 DFS 推流都只沿残量容量大于 0 的边前进，因此得到的新流仍然满足容量约束和流量守恒。

如果某次 BFS 后 t 不可达，把所有在残量网络中能从 s 到达的点记为集合 S，其余点为 T。因为没有残量边能从 S 继续穿到 T，所以跨割的正向边都已经满流；反向能撤回的边不会增加从 S 到 T 的容量。于是当前流量等于这个割的容量。

任意可行流 <= 任意割容量 当前流量 = 某个割容量 所以：当前流就是最大流，这个割就是最小割

分层图和阻塞流影响的是效率，而不是正确性的根基。即使每轮只找一条路径，只要合法增广并在无路时停止，仍然是最大流；Dinic 只是更系统地批量处理当前最短层次中的多条路径。

七、复杂度直觉

Dinic 在一般图上的经典复杂度常写作 O(V^2 E)。可以从两个层面理解：

部分直觉代价
BFS 分层扫描残量边，计算每个点的层数。O(E)
一轮阻塞流DFS 在当前弧优化下推进，失败边不会被无限重复扫描。常按 O(VE) 上界理解
轮数每轮阻塞当前最短层次结构，下一次若还可达，最短距离会向后推进。不超过 O(V) 轮

在二分图匹配这类特殊网络上，Dinic 的表现更好，常见上界可到 O(E sqrt(V)) 量级，与 Hopcroft-Karp 的分层批量增广思想非常接近。

实践中 Dinic 往往比 Edmonds-Karp 更常用：它不会受最大流数值 F 直接控制，在稀疏图、整数容量和多路径并存的网络里通常很稳。

八、实现要点

边结构

每次加边都同时加一条反向边。原边初始容量为 cap，反向边初始容量为 0；推流时正边减、反边加。

BFS

只访问残量容量大于 0 的边。若 t 没有层数，本轮主循环结束，当前流就是最大流。

DFS

只能走 level[v] = level[u] + 1 的边。返回值为本次实际推过去的流量，返回 0 表示这条分支走不通。

当前弧

it[u] 要在每轮 BFS 后重置。它属于当前分层图，一旦层次重建，旧的扫描位置就不再可靠。

add_edge(u, v, cap):
    graph[u].push({to: v, rev: graph[v].size, cap: cap})
    graph[v].push({to: u, rev: graph[u].size - 1, cap: 0})

push(edge, x):
    edge.cap -= x
    graph[edge.to][edge.rev].cap += x
