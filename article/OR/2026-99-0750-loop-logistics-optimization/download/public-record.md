Publicly visible abstract summary:

The paper proposes LOOP, a Language-Model Orchestrated Optimization Pipeline for logistics optimization. The core problem is that LLMs can provide a natural-language interface but often fail when asked to directly generate reliable mathematical constraints and executable optimization code. LOOP uses task-specific agents, semantic mapping, ensemble difference analysis, and multi-layered verification to translate changing logistics requirements into solver-oriented code while preserving optimization-model rigor.

Reported evaluation:

- Task: capacitated vehicle routing problem (CVRP)
- Dataset: Augerat benchmark
- Scale: 54 cases across six difficulty levels
- Reported code pass rate: 96.3%
- Reported solution gap from expert baselines: 1.5%
